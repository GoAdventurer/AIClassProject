## 1. 项目概述

一个**单文件网页版围棋游戏**：双击 `index.html` 即可在浏览器中游玩，无需服务器、无需安装。视觉采用「暗夜霓虹」风格——深蓝渐变背景，发光网格与棋子。支持 9×9 / 13×13 / 19×19 三种棋盘，人机与双人对战，三档 AI 难度，悔棋无限次，落子/提子等音效，棋局可导出/导入 SGF 标准棋谱并回放。

### 1.1 关键决策（已经过用户确认）

| 决策点 | 选择 |
|---|---|
| 项目位置 | `~/projects/go-game/` |
| 棋盘大小 | 同时支持 9×9 / 13×13 / 19×19 |
| 规则 | 中国规则（贴目 7.5、数子法、全局同形禁止） |
| AI 实现 | 纯 JS：低=随机+约束 / 中=启发式+小搜索 / 高=MCTS |
| 视觉风格 | 暗夜霓虹（深蓝 + 蓝/青色发光） |
| 棋盘渲染 | Canvas 2D（三层叠加） |
| 音效 | 内置真实音效（CC0 资源，base64 内联） |
| 棋谱格式 | SGF 标准格式 |
| 工程框架 | 纯静态：单个 HTML（一切内联，可双击运行） |
| 高级玩法 | 推演 + 胜率提示 + 自动数子/划领地 + 计时器 |
| 悔棋 | 不限次数 |
| 死子判定 | AI 自动判定（保留手动覆盖按钮作兜底） |

## 2. 整体架构

单页项目，运行时分为 **5 个独立模块**，仅通过 EventBus 通信，互不持有引用：

```
              ┌──────────────────────┐
              │      EventBus        │   全局事件中枢
              └──────────────────────┘
                ▲    ▲    ▲    ▲    ▲
        ┌───────┘    │    │    │    └───────┐
        │            │    │    │            │
 ┌─────────┐  ┌──────────┐  ┌────┐  ┌────────┐  ┌─────────┐
 │GoEngine │  │ AIPlayer │  │View│  │AudioFX │  │ SgfIO   │
 │规则/状态│  │三档难度  │  │三层│  │Web     │  │导入导出 │
 │         │  │分片让出  │  │Canv│  │ Audio  │  │         │
 └─────────┘  └──────────┘  └────┘  └────────┘  └─────────┘
```

- **GoEngine**：纯函数化的规则核心；无 DOM 依赖，可被单元测试直接调用
- **AIPlayer**：三档难度，**主线程内分片执行**（MCTS 用 rAF 让出，UI 不卡）
- **View**：三层 Canvas（背景/棋子/特效），高 DPR 适配
- **AudioFX**：4 个内置音效（base64 内联），Web Audio API 播放
- **SgfIO**：SGF 序列化/解析，回放器把 SGF 重新喂给 GoEngine

**核心数据流**：
1. 用户点棋盘 → View 发 `move-attempt`
2. GoEngine 校验 → 通过则更新棋局并发 `move-applied`
3. View 重绘 + AudioFX 播音
4. AIPlayer（如适用）收到事件后异步算下一步，分片执行
5. 算完发 `ai-move`，回到第 2 步

**为什么不用 Web Worker**：用户要求双击 `index.html` 即玩，Worker 需要单独 JS 文件，与单文件部署冲突。AI 改为主线程分片执行（rAF 让出），UI 帧率仍可保持 ≥30fps。

## 3. 模块详细设计

### 3.1 GoEngine（规则核心）

**核心数据结构**（不可变快照风格）：

```js
GameState = {
  size: 9 | 13 | 19,
  board: Int8Array,                    // size*size，0=空 1=黑 2=白
  toPlay: 1 | 2,
  captures: { black: int, white: int },
  history: GameState[],                // 用于悔棋 + 全局同形检测
  moveLog: Move[],                     // 用于 SGF 导出
  passes: int,                         // 连续 pass 数（==2 终局）
  komi: 7.5,
  phase: "playing" | "scoring" | "ended"
}

Move = { color: 1|2, x: int, y: int } | { color, pass: true }
```

**关键 API**：

| 函数 | 行为 |
|---|---|
| `createGame(size, komi)` | 新建对局 |
| `tryMove(state, x, y)` | 返回 `{ ok: true, next }` 或 `{ ok: false, reason: "occupied"\|"suicide"\|"ko" }` |
| `undo(state)` | 弹出最近一步快照 |
| `pass(state)` | 标记 pass；连续两次进 scoring |
| `resign(state, color)` | 直接定胜负 |
| `score(state)` | 中国数子：活子 + 单方围地 + 贴目 |
| `detectDeadStones(state)` | 自动判死，返回死子坐标列表 |

**快照而非 patch 的理由**：悔棋一行 `history.pop()` 解决；劫争用 `Map<zobristHash, count>` 检测全局同形。19×19 满盘 360 步 ≈ 130KB，可接受。

**Zobrist 哈希**：初始化时为每格 × 颜色随机一个 64-bit；落子/提子异或；用于劫争同形检测。

### 3.2 AIPlayer（三档难度）

**总入口**：

```js
AIPlayer.think(state, level, onProgress) → Promise<Move>
```

`onProgress(0~1)` 回调用于 MCTS 进度条；内部用 `requestAnimationFrame` 让出主线程。

**三档实现**：

| 难度 | 算法 | 思考时间 | 大致水平 |
|---|---|---|---|
| **低** | 随机选合法点 + 4 条约束：①不下自己眼位 ②对手有"提我"威胁时优先逃 ③对手有≥4 子相连可吃时去吃 ④否则随机 | <50ms（同步） | 新手友好 |
| **中** | 启发式估值（influence map + 实地估算 + 厚度），对前 20 候选点做 1-2 层 alpha-beta | 200-500ms（分片） | 业余 10K |
| **高** | MCTS：UCB1 选点 + 启发式 rollout + 反向更新；模拟次数 800（9路）/ 1500（13路）/ 2000（19路） | 1-3 秒（分片 + rAF） | 业余 5K~15K |

**让出主线程策略**：
- 低：直接同步算（<50ms 不会感知）
- 中：每 50ms 用 `setTimeout(0)` 让出一次
- 高：每完成 N=50 次 simulation 用 `requestAnimationFrame` 让出，UI 同时更新进度条

**取消机制**：用户悔棋时设 `cancelToken`，AI 在让出点检查并立即返回。

### 3.3 View（Canvas 渲染）

**三层 Canvas 叠在同一容器内**：

| 层 | 内容 | 重绘时机 |
|---|---|---|
| `boardLayer` | 深蓝渐变背景、发光网格线、星位、坐标 | resize 或换棋盘大小 |
| `stoneLayer` | 所有棋子 | 落子/提子时局部重绘对应区域 |
| `fxLayer` | 落子涟漪、最后一手脉冲、鼠标悬停虚影、提子消散、胜率/推演提示 | rAF 持续驱动（仅有动画时） |

**棋子样式（暗夜霓虹）**：
- 黑子：深蓝径向渐变（中心 #6080ff → 边 #000）+ 蓝色光晕
- 白子：浅蓝径向渐变（中心 #fff → 边 #6090c0）+ 青色光晕
- 最后一手：脉冲动画 1.5 秒循环

**响应式**：`ResizeObserver` 监听容器大小变化，按 `min(w, h)` 计算单格 px；`devicePixelRatio` 适配高清屏。

### 3.4 AudioFX

4 个 CC0 音效（来源 freesound.org），内联为 base64 data URI：

| 资源 | 触发 | 体积 |
|---|---|---|
| `stone-place.ogg` | 落子 | ~20KB |
| `stone-capture.ogg` | 提子 | ~25KB |
| `game-start.ogg` | 对局开始 | ~25KB |
| `game-win.ogg` | 胜利/认输 | ~30KB |

用 Web Audio API（不是 `<audio>`）：解码后缓存 AudioBuffer，播放零延迟、可并发。设置面板：总开关 + 4 个独立开关 + 音量条。

### 3.5 SgfIO

**导出**：把 `moveLog` 序列化为标准 SGF：

```
(;FF[4]GM[1]SZ[19]KM[7.5]RU[Chinese]
;B[pd];W[dp];B[pp];W[dd];...
;C[undo]                            # 悔棋点用注释标记
)
```

**导入**：
1. Tokenize → 解析为 properties
2. 重放每一步到 GoEngine（异常 SGF 中途有违规手时，停止并提示用户从第 K 手起决定继续/丢弃）
3. 得到完整 `history`，回放控件即可前后跳转

**回放控件**：`⏮ 首手 / ◀ 上一手 / ▶/⏸ 播放 / ▶ 下一手 / ⏭ 末手 / [速度: 0.5x 1x 2x 4x]`，进度滑块支持拖动跳转，左右方向键单步。

## 5. 高级玩法

### 5.1 悔棋

- **双人模式**：`history.pop()` 一次，回到上一手
- **人机模式**：连续 `pop()` 两次（撤掉 AI 回应 + 自己一步）；AI 思考中时先 cancel
- 次数**不限**
- 悔棋点用 SGF 注释 `C[undo]` 记录，导出时保留

### 5.2 推演模式（仅双人）

- 按 `⏵ 推演` 进入沙盒分支，玩家可继续落子试看效果
- 顶部出现红色横幅 `推演中 · 不影响主棋谱`，右下角 `退出推演`
- 推演子用半透明显示
- 退出时回到分支起点

### 5.3 胜率提示（仅 AI 高难度）

- MCTS 自然产出胜率 `wins / visits`
- 对局中实时显示当前局面黑方胜率条：`◤◤◤◤◇◇◇◇◇◇  41%`
- 鼠标悬停在合法点 ≥0.5 秒触发"假设落子后胜率"查询：`如果黑下这里 → 48%`（防抖避免狂查）

### 5.4 计时器

- 双方独立计时，**仅累计无限制**
- 落子时切换计时器；pass/认输停表
- 暂停按钮双方都停
- 回放和推演不计时

### 5.5 自动数子 / 划领地

- 双方连续 pass → `phase: scoring`
- **AI 自动判死**：对每个棋子串，若被对方棋子+对方领地完全包围且自身无法形成两眼，判死
- 棋盘渲染：
  - 死子：半透明 X 标记
  - 黑领地：半透明深蓝色块
  - 白领地：半透明浅蓝色块
- 底部结算：`黑 41 + 提子 5 = 46  /  白 36 + 7.5 贴目 = 43.5  →  黑胜 2.5 目`
- **兜底**：结果页有小按钮 `不服？手动确认死子`，进入手动模式可逐子切换"活/死"，双方按"确认"后定胜负

### 5.6 回放

- 任何对局结束后或导入 SGF 时进入回放模式
- 控件：见 §3.5
- 支持"跳出回放回到当前局面继续下"——会把回放点之后的棋删掉作为新对局起点

## 7. 错误处理

| 场景 | 处理 |
|---|---|
| 用户点空白处 / 已落子点 | 静默忽略 |
| 用户落违规子（劫争/自杀） | 该点闪红 + Toast `劫争禁同形` / `禁手：自杀` |
| AI 找不到合法着法 | AI 主动 pass |
| AI 计算异常（罕见） | catch 后 AI pass，控制台 warn |
| 导入的 SGF 格式错误 | 弹错对话框 `棋谱解析失败：第 N 处错误`，不破坏当前棋局 |
| 导入的 SGF 中含违规手 | 显示已解析到第 K 手，让用户决定继续/丢弃 |
| localStorage 满或被禁 | 静默降级，仅一次提示 `自动存档不可用` |
| AI 思考中刷新页面 | 自动存档已保留落子前状态，刷新后从用户上一手继续 |

## 8. 测试策略

### 8.1 单元测试（`tests/runner.html`，浏览器内跑）

1. **GoEngine 规则**
   - 基础落子合法性（占位、超界）
   - 提子（角/边/中央，单子/连块）
   - 自杀禁止 + 自杀但能反提的情况应允许
   - 劫争同形禁止（含 3 步循环、打两手劫的边界）
   - 数子结果与 5 个人工标定 9×9 终局对比

2. **SGF 序列化**
   - 导出后再导入应得到同一棋局
   - 解析 KGS / OGS 真实样本（`samples/` 中放 3-5 个）
   - 异常输入不崩溃

3. **AI 合法性**
   - 三档 × 三种棋盘各跑 100 局自我对弈不抛错
   - AI 永远只产出合法着法

4. **数子算法**
   - 5 个手工标定终局，自动判死结果应与人类一致

### 8.2 手动验收清单（README 写出，发布前必跑）

- [ ] 9 / 13 / 19 三种棋盘开局正常
- [ ] 三档 AI 各下 5 步无明显怪招
- [ ] 悔棋 5 次还能正常继续
- [ ] 推演进入/退出后棋局完好
- [ ] 导出 SGF 可在 GoGui / OGS 中打开
- [ ] 静音/取消静音生效
- [ ] 移动端竖屏可玩 9×9

## 10. 自动存档与恢复

- 每次落子后写 `localStorage["goneon.current"]`：当前 SGF + phase + AI 配置
- 启动时检测：有存档 → 主菜单底部出现 `继续上一局 (黑 第 12 手)` 按钮
- 对局结束清空 `goneon.current`，把棋谱追加到 `localStorage["goneon.history"]`（保留最近 10 局）

## 12. 已知风险与缓解

| 风险 | 缓解 |
|---|---|
| 19×19 MCTS 在主线程跑 3 秒会感知卡顿 | 严格分片 + rAF 让出 + 进度条；用户可降到中难度 |
| 19×19 自动判死复杂残局可能误判 | 提供"手动确认死子"兜底按钮 |
| SGF 实现细节庞杂（变体、注释、扩展属性） | 仅支持核心子集（FF/GM/SZ/KM/RU/B/W/C），声明在 README |
| 单文件 350KB 首次加载体积 | 双击场景下不是问题；如部署到服务器，可后续拆分 |
| 不同浏览器 Web Audio 行为差异（首次播放需用户交互） | 第一次用户点击棋盘时初始化 AudioContext |
