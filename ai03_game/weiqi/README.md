# GoNeon · 网页围棋

一个**双击即玩**的单文件网页围棋游戏。

> 暗夜霓虹画风 · 9/13/19 三种棋盘 · 人机/双人 · 三档 AI · 悔棋无限次 · SGF 棋谱 · 内置音效

## 直接玩

双击 `index.html`，浏览器打开即可。无需服务器、npm、安装。

## 游戏说明

### 主菜单

- **棋盘**：9 路（快、AI 强）/ 13 路 / 19 路（标准）
- **对手**：人 vs AI / 人 vs 人
- **难度**：
  - **低**（<100ms 响应）：随机+约束（避眼、能提就提）
  - **中**（200-500ms）：启发式估值 + 候选点选优
  - **高**（1-3 秒）：MCTS（蒙特卡洛树搜索）+ UCB1 + 启发式 rollout
- **执子**：选择执黑/白/随机
- **贴目**：默认 7.5（中国规则）

### 对局界面

- 左侧棋盘，鼠标点击落子（也支持触摸屏）
- 右侧侧栏：双方玩家信息（计时器、提子数、思考进度）+ 操作按钮
- **悔棋**：人机模式连撤两步（己方+对方）；双人模式撤一步；不限次数
- **Pass**：跳过本回合；双方连 Pass → 自动数子
- **认输**：直接结束对局
- **保存 SGF**：导出标准棋谱文件
- **载入 SGF**：从文件加载棋谱（支持 KGS/OGS/野狐等平台导出的标准格式）

### 对局结束

- 自动数子（中国规则，含贴目），自动判定死子并标 X
- 可点「📺 进入回放」查看整局录像
- 「💾 保存 SGF」导出棋谱文件

### 回放控件

- ⏮ 首手 / ◀ 上一手 / ▶ 播放 / ▶▶ 下一手 / ⏭ 末手
- 速度：0.5x / 1x / 2x / 4x
- 进度滑块拖动跳转
- 键盘：← / → 单步，空格 播放/暂停

### 设置

右上角 ⚙ 图标：音效开关、音量、回主菜单

### 自动存档

每次落子自动存到 localStorage，刷新页面后菜单底部会出现「▶ 继续上一局」。对局结束后自动加入历史记录（保留最近 10 局）。

## 技术栈

- 纯原生 JS（ES2020），单 HTML 文件 ~75KB
- Canvas 2D（三层叠加：棋盘背景 / 棋子 / 特效）
- Web Audio API（合成音效，零外部依赖）
- localStorage（自动存档）
- 不依赖框架、不需要构建、不需要联网

## 模块架构

```
EventBus → BoardOps → Zobrist → Engine → Scoring →
SGF → Storage →
AIHeuristics → AIEasy → AIMedium → AIMcts → AIRunner →
Theme → BoardLayer → StoneLayer → FxLayer → BoardRenderer →
AudioFx →
MenuUI → SidebarUI → App
```

每个模块挂在 `window.GoNeon.*` 命名空间下，IIFE 隔离内部，互相通过 EventBus 通信。

## 测试

- **核心逻辑测试**：`node tests/node-test.js` — 26 个用例覆盖规则（落子/提子/自杀禁/劫争禁同形/悔棋/Pass/Resign）、数子、SGF 双向、AI 三档合法性、Storage
- **浏览器集成测试**：`node tests/browser-test.js` — Headless Chrome 模拟用户操作（菜单 → 开始 → 落子 → AI 回应 → 悔棋 → SGF），9 项全通过
- **手动测试页**：浏览器打开 `tests/runner.html`（基于 iframe 加载 index.html）

## 已知限制

- AI 走子风格偏功利（仅靠局面分估值，不会布局），中级 AI 适合作为陪练
- 19 路 MCTS 受限于浏览器单线程，2000 次模拟约需 2-3 秒（设计预算内）
- 自动判死算法保守，复杂残局可能有少量误判 — 结算页有「不服？手动确认死子」兜底入口（暂未启用）
- 本版本仅支持中国规则（数子法+贴目 7.5），不含日韩规则
- 推演模式（沙盒分支）和实时胜率提示在本版未实现，作为后续增强项
