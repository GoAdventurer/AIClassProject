门票助手机器人代码逻辑整理

此文档整理了 assistant_ticket_bot-1.py 的代码逻辑结构与功能分析。

1. 项目概述

这是一个智能门票助手机器人，主要功能包括：

  • 提供门票订单查询服务
  • 支持SQL查询工具，可对门票订单数据进行查询分析
  • 支持终端交互（TUI）和图形界面（GUI）两种模式
  • 基于Qwen Agent框架开发

2. 核心组件

2.1 配置模块

import os import asyncio from typing import Optional import dashscope from qwen_agent.agents import Assistant from qwen_agent.gui import WebUI import pandas as pd from sqlalchemy import create_engine from qwen_agent.tools.base import BaseTool, register_tool # 定义资源文件根目录 ROOT_RESOURCE = os.path.join(os.path.dirname(__file__), 'resource') # 配置 DashScope dashscope.api_key = os.getenv('DASHSCOPE_API_KEY', '') # 从环境变量获取 API Key dashscope.timeout = 30 # 设置超时时间为 30 秒

2.2 系统提示词 (System Prompt)

定义了数据库表结构和助手角色：

  • 表名：tkt_orders（门票订单表）
  • 字段：包含订单时间、用户ID、身份证号、性别、年龄、省份、SKU、商品ID、订单ID、销售渠道、状态、金额、数量等
  • SKU分类：一日门票和二日门票的不同类型

2.3 SQL执行工具 (exc_sql)

注册为可调用函数，用于执行SQL查询：

  • 接收SQL语句作为输入参数
  • 连接MySQL数据库执行查询
  • 返回前10行查询结果

6. 主要功能函数

6.1 init_agent_service()

初始化门票助手服务：

  • 配置LLM参数（qwen-turbo-latest模型）
  • 创建Assistant实例
  • 设置系统提示词和可调用函数

6.2 app_tui()

终端交互模式：

  • 循环获取用户输入
  • 支持文本和文件输入
  • 维护对话历史
  • 调用助手处理并显示结果

6.3 app_gui()

图形界面模式：

  • 启动Web UI界面
  • 提供示例问题建议
  • 实现可视化交互

7. 运行流程

sequenceDiagram participant U as 用户 participant B as 门票助手 participant DB as 数据库 participant LLM as 语言模型 U->>B: 输入问题 B->>LLM: 解析问题 alt 需要SQL查询 LLM->>B: 生成SQL语句 B->>DB: 执行SQL DB-->>B: 返回查询结果 B->>LLM: 处理结果 else 直接回答 LLM-->>B: 直接生成答案 end B-->>U: 返回答案

8. 代码特点

  • 模块化设计：将工具、系统提示、主逻辑分离
  • 可扩展性：通过函数调用机制扩展功能
  • 双模式支持：终端TUI和Web GUI界面
  • 安全配置：API Key从环境变量获取
  • 错误处理：包含异常捕获和错误信息返回
